# Design System

## Overview

The design system follows a calm, elderly-friendly philosophy. Every visual choice prioritizes readability, patience, and emotional warmth.

## Color Palette

| Token | Hex | Usage |
|-------|-----|-------|
| background | #FAF8F5 | Page background, warm white |
| primary | #3A5A40 | Buttons, progress, primary actions |
| accent | #D4A573 | Highlights, warm accent |
| card | #FFFFFF | Card backgrounds |
| text | #2D2D2D | Primary text |
| textLight | #8A8A8A | Secondary text |
| divider | #E8E4E0 | Borders, separators |

## Typography

| Style | Size | Weight | Usage |
|-------|------|--------|-------|
| display | 40px | 700 | App title, hero text |
| heading | 28px | 600 | Section headers |
| question | 34-38px | 500 | Questions (primary interaction) |
| body | 22px | 400 | Body text, descriptions |
| button | 22px | 600 | Button labels |
| caption | 18px | 400 | Hints, secondary info |

## Touch Targets

- Minimum: 60px (elderly-friendly)
- Recommended: 72px
- Button height: 68-80px

## Border Radius

| Token | Value |
|-------|-------|
| s | 8px |
| m | 14px |
| l | 18px |
| xl | 24px |

## Animation

- Duration: 250-300ms
- Curve:.easeOut
- No jarring transitions
- Gentle fade + slide

## Principles

1. One job per screen
2. Maximum 3 actions visible
3. Huge white space
4. No hamburger menus
5. No bottom navigation
6. Portrait only
7. No long-press interactions
8. Warm, grandchild-like language
