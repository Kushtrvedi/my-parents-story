# Security Policy

## Reporting Vulnerabilities

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** open a public GitHub issue
2. Email security concerns to the maintainers
3. Include a description of the vulnerability
4. Include steps to reproduce
5. Allow reasonable time for a fix before public disclosure

## API Key Protection

- OpenAI API keys are stored in `.env` files (never committed)
- Firebase configuration is generated by `flutterfire configure`
- API keys are never hardcoded in source code
- Use environment variables or secure storage in production

## Firebase Security Rules

### Firestore Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can only access their own data
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    
    match /parent_profiles/{profileId} {
      allow read, write: if request.auth != null && 
        resource.data.userId == request.auth.uid;
    }
    
    match /responses/{responseId} {
      allow read, write: if request.auth != null;
    }
    
    match /generated_chapters/{chapterId} {
      allow read, write: if request.auth != null;
    }
  }
}
```

### Storage Rules

```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /users/{userId}/{allPaths=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

## Data Privacy

- User data is stored in their personal Firebase project
- No data is shared between users
- Voice recordings are processed locally when possible
- Photos are stored in user-specific Firebase Storage paths
- All data can be deleted by the user

## Authentication

- Firebase Anonymous Authentication for initial access
- No personally identifiable information required to start
- Optional account creation for data persistence

## Best Practices

- Keep dependencies updated
- Use latest stable Flutter version
- Never commit `.env` files
- Use `flutterfire configure` for Firebase setup
- Review Firebase security rules regularly
- Enable Firebase App Check in production

## Dependencies

We regularly audit dependencies for vulnerabilities:

- `flutter pub audit` for Dart/Flutter packages
- GitHub Dependabot for automated updates
- Manual review of critical dependencies
